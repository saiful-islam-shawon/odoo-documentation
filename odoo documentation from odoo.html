Inverse Function:

from odoo import api, fields, models

class TestComputed(models.Model):
    _name = "test.computed"

    total = fields.Float(compute="_compute_total", inverse="_inverse_total")
    amount = fields.Float()

    @api.depends("amount")
    def _compute_total(self):
        for record in self:
            record.total = 2.0 * record.amount

    def _inverse_total(self):
        for record in self:
            record.amount = record.total / 2.0


COMPUTE (reading)
─────────────────
price + tax  →  _compute_total()  →  total is calculated


INVERSE (writing)
─────────────────
user edits total  →  _inverse_total()  →  price is updated

Real world example

Price = 1000
Tax   = 10%
─────────────────────────
Total = 1100   ← computed automatically


Now user manually changes Total to 2200:
─────────────────────────
_inverse_total() runs
Price = 2200 / 1.10 = 2000  ← price updated automatically
Tax   = 10%                  ← stays same

Constraints: how can we prevent users from entering incorrect data? For example, in our real estate module nothing prevents users from setting a negative expected price.

from odoo import fields, models, api
from odoo.exceptions import ValidationError

class EstateProperty(models.Model):
    _name = "estate.property"

    name = fields.Char("Name")
    price = fields.Float("Price")
    date = fields.Date("Date")

    @api.constrains('price')
    def _check_price(self):
        for record in self:
            if record.price <= 0:
                raise ValidationError("Price must be greater than 0!")

    @api.constrains('price', 'date')
    def _check_price_and_date(self):
        for record in self:
            if record.price < 1000:
                raise ValidationError("Price must be at least 1000!")
            if not record.date:
                raise ValidationError("Date is required!")

Order field: The _order attribute takes a string containing a list of fields which will be used for sorting. It will be converted to an order_by clause in SQL. For example:

from odoo import fields, models

class TestModel(models.Model):
    _name = "test_model"
    _description = "Test Model"
    _order = "id desc"

    description = fields.Char()

Our records are ordered by descending id, meaning the highest comes first.

from odoo import fields, models

class InheritedModel(models.Model):
    _inherit = ["inherited.model"]

    new_field = fields.Char(string="New Field")

<record id="inherited_model_view_form" model="ir.ui.view">
    <field name="name">inherited.model.form.inherit.test</field>
    <field name="model">inherited.model</field>
    <field name="inherit_id" ref="inherited.inherited_model_view_form"/>
    <field name="arch" type="xml">
        <!-- find field description and add the field new_field after it -->
        <xpath expr="//field[@name='description']" position="after">
          <field name="new_field"/>
        </xpath>
        <!-- Modify existing field -->  
		<xpath  expr="//field[@name='description']"  position="attributes">  
			<attribute  name="readonly">1</attribute>  
			<attribute  name="string">New Description</attribute>  
		</xpath>
    </field>
</record>
